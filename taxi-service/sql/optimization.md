# Оптимизация SQL запросов для Такси-сервиса

## 1. Индексы и их назначение

### Таблица `users`

| Индекс | Тип | Назначение | Используемые запросы |
|--------|-----|------------|---------------------|
| `idx_users_login` | B-Tree | Быстрый поиск пользователя по логину | Поиск пользователя по логину, аутентификация |
| `idx_users_name` | B-Tree | Поиск по маске имени/фамилии | Поиск пользователей по маске |
| `idx_users_email` | B-Tree | Проверка уникальности email | Создание пользователя |

### Таблица `drivers`

| Индекс | Тип | Назначение | Используемые запросы |
|--------|-----|------------|---------------------|
| `idx_drivers_status` | B-Tree | Поиск свободных водителей | Принятие заказа, создание заказа |
| `idx_drivers_car_number` | B-Tree | Проверка уникальности номера | Регистрация водителя |

### Таблица `rides`

| Индекс | Тип | Назначение | Используемые запросы |
|--------|-----|------------|---------------------|
| `idx_rides_user_id` | B-Tree | JOIN с users | История поездок пользователя |
| `idx_rides_driver_id` | B-Tree | JOIN с drivers | Поиск поездок водителя |
| `idx_rides_status` | B-Tree | Фильтрация по статусу | Активные заказы |
| `idx_rides_created_at` | B-Tree DESC | Сортировка по дате | Активные заказы, история |
| `idx_rides_user_status` | B-Tree | Составной поиск | Фильтрация поездок пользователя по статусу |
| `idx_rides_driver_status` | B-Tree | Составной поиск | Поиск поездок водителя по статусу |
